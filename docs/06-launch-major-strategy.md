# 06. 초기 런칭 전공군 전략

문서 목적: 물리치료, 작업치료, 방사선, 임상병리 4개 전공군을 동시에 시작할 때 어떤 구조와 우선순위가 초기 활성화에 가장 유리한지 정리한다.

## 가정 (Assumptions)

- 초기 전공군 4개는 동시에 시작한다.
- 이 4개는 서로 완전히 다른 커뮤니티가 아니라 공통 고민이 많은 보건의료기사 계열 네트워크로 묶을 수 있다.

## 왜 4개 전공군을 동시에 시작하는가

### 1. 모수 확보

한 전공군만으로 시작하면 콘텐츠와 활동량이 적어 초기 빈 게시판 문제가 더 심해질 수 있다. 4개 전공군을 동시에 묶으면 최소한의 피드 밀도를 만들 가능성이 높아진다.

### 2. 공통 주제의 중첩

물리치료, 작업치료, 방사선, 임상병리는 아래 주제가 상당 부분 겹친다.

- 실습
- 국가시험
- 병원 취업
- 스터디
- 팀모집
- 질문답변

즉 전공은 다르지만, 정보 구조를 통합하기 좋은 공통 축이 분명하다.

### 3. 니치하지만 너무 좁지는 않은 전략

대학생 전체 커뮤니티보다 좁지만, 단일 학과 앱보다는 넓다. 이 중간 폭이 MVP 초기에 더 현실적이다.

## 초기 구조 원칙

1. 4개 전공군을 하나의 상위 네트워크 안에서 보여준다.
2. 홈과 모집은 통합 구조를 우선으로 한다.
3. 전공군 분리는 MVP에서는 홈 내부 필터와 집중 보기 수준으로 둔다.
4. 학교 보드는 보조 가치로 제한한다.

## 추천 구조

### 상위 커뮤니티

- 이름상으로는 "보건의료기사 계열 전공 네트워크"에 해당하는 하나의 상위 공간
- 홈에서는 4개 전공군의 글이 함께 섞여 보인다

### 전공군 하위 구분

- 홈 상단 필터 또는 전공군 집중 보기 상태에서 4개 전공군 선택
- 각 전공군 안에서 실습, 국시, 취업, 질문답변, 자유, 모집 카테고리를 공유

### 학교 보드

- 같은 학교 인증 사용자만 접근 가능
- 학교 생활 질문, 수업/동아리/행정성 질문 등 제한형 용도로 유지

## 빈 게시판 문제를 줄이는 방법

### 1. 통합 홈 우선 노출

첫 화면은 "물리치료 게시판", "작업치료 게시판"처럼 나뉜 목록이 아니라 통합 인기글/최신글이어야 한다.

### 2. 통합 모집 우선

모집은 전공군별 분리보다 전체 리스트가 먼저 보여야 한다. 이후 필터로 좁히는 방식이 더 밀도가 높다.

### 3. 카테고리 중심 피드

사용자는 "실습 정보"나 "국시 정보"를 찾을 가능성이 높다. 따라서 콘텐츠 분류는 전공군만이 아니라 카테고리 중심 축도 가져야 한다.

### 4. 운영 초기 콘텐츠 시딩

운영 측면에서도 아래 준비가 필요하다.

- 전공군별 자주 묻는 질문 초안 5~10개
- 실습/국시/취업 카테고리 예시 글
- 모집글 템플릿
- 신고 기준 공지

## 통합 홈 / 전공별 보드 / 학교 보드 우선순위

1. 통합 홈
2. 통합 모집
3. 전공군 필터 피드
4. 학교 보드

이 우선순위가 중요한 이유는 "콘텐츠 밀도"를 먼저 만들고, 그다음 "세부 소속감"을 붙여야 하기 때문이다.

## 왜 이 4개 조합이 초기 활성화에 유리한가

- 모두 보건의료기사 계열이라는 공통 정체성이 있다.
- 실습과 취업을 병원/의료기관 맥락으로 공유할 수 있다.
- 학과별 질문은 다르더라도 정보의 큰 주제는 겹친다.
- 사용자가 느끼는 "같은 분야 사람들끼리 모여 있다"는 감각을 만들기 좋다.

## 5번째, 6번째 전공 확장 기준

확장은 아래 기준을 만족할 때만 진행한다.

1. 기존 4개 전공군의 통합 홈 활동성이 유지될 것
2. 새 전공군이 기존 카테고리 구조와 어느 정도 공통점을 가질 것
3. 운영 정책과 인증 정책을 새 전공군에 무리 없이 적용할 수 있을 것
4. 초기 시딩 콘텐츠를 함께 준비할 수 있을 것

후보 확장 시 고려 기준:

- 국가시험/실습/취업 구조의 유사성
- 병원/의료기관 커리어 맥락의 유사성
- 팀모집 및 정보교류 수요의 크기

## 4개 전공군 동시 베타 Go / No-Go 기준

아래 기준을 만족하지 못하면 4개 전공군 동시 오픈보다, 베타 규모를 줄이거나 초청형으로 여는 쪽이 안전하다.

| 항목 | 최소 기준 | 판단 이유 |
| --- | --- | --- |
| 전공군별 시딩 글 수 | 전공군당 10개 이상 | 특정 전공군만 비어 보이는 문제 방지 |
| 통합 모집 예시 | 전체 8개 이상 | 모집 탭 첫 인상 확보 |
| 인증 가능 학교 수 | 이메일 또는 수동 인증 가이드가 준비된 15개교 이상 | 인증 이탈 최소화 |
| 전공군별 베타 대기 인원 | 전공군당 8명 이상 | 동시 시작 최소 밀도 확보 |
| 운영 담당자 | 1차 검토자 1명 + 백업 1명 | 수동 인증과 신고 대응 가능성 확보 |

이 기준은 마케팅 목표가 아니라 "오픈해도 너무 텅 비어 보이지 않을 최소선"으로 본다.

## 추천 런칭 메시지 방향

- 같은 학교가 아니어도, 같은 계열이면 통한다
- 실습, 국시, 취업, 팀모집을 한곳에서
- 학교 인증 기반으로 더 믿을 수 있는 전공 네트워크

## 결론

4개 전공군 동시 런칭은 범위를 넓히는 전략이 아니라, 오히려 초기 밀도를 확보하기 위한 전략으로 봐야 한다. 핵심은 이 4개를 각각 따로 운영하는 것이 아니라, 하나의 상위 네트워크 안에서 통합 홈과 통합 모집으로 묶고 전공군 필터를 붙이는 데 있다.
