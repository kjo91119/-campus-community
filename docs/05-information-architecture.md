# 05. 정보구조(IA)

문서 목적: MVP 기준으로 어떤 네비게이션과 화면 구조가 가장 적절한지 정하고, 4개 전공군 동시 런칭에서도 게시판이 비어 보이지 않게 만드는 구조를 제안한다.

## 가정 (Assumptions)

- MVP는 정보 밀도 확보가 가장 중요하다.
- 전공군별 완전 분리보다 통합 홈 중심 구조가 초기 활성화에 더 유리하다.

## 이번 라운드에서 고정한 결정

- MVP 바텀탭은 `홈 / 모집 / 학교 / 프로필` 4개로 고정한다.
- 전공군 탐색은 별도 바텀탭이 아니라 홈 내부의 전공군 필터와 전공군 집중 보기 상태로 시작한다.
- 전공 전용 탭은 V1 이후, 홈 내 전공군 필터 사용률과 콘텐츠 밀도가 충분할 때 다시 검토한다.

## IA 설계 원칙

1. 첫 진입에서 바로 읽을 글이 보여야 한다.
2. 전공군별 구분은 필요하지만, 첫 화면부터 섬처럼 쪼개지지 않아야 한다.
3. 학교 보드는 존재하되 주력 체류 공간이 되어서는 안 된다.
4. 모집은 게시글 하위 기능이 아니라 독립 탐색 니즈가 있으므로 별도 진입점을 둔다.
5. 알림은 필요하지만 바텀탭 한 자리를 차지할 정도로 MVP 우선순위가 높지는 않다.

## 구조 후보 비교

### 후보 A. 홈 / 전공 / 학교 / 모집 / 알림 / 프로필

장점:

- 역할 구분이 가장 명확하다.
- 확장성이 좋다.

단점:

- 탭이 많아진다.
- 초기에는 전공과 홈의 차이가 애매해질 수 있다.
- 알림 탭은 초기에 비어 보일 가능성이 크다.

### 후보 B. 홈(전공군 필터 포함) / 모집 / 학교 / 프로필

장점:

- 가장 단순하다.
- 전공 콘텐츠를 홈 안의 필터로 통합할 수 있다.
- 빈 화면을 줄이기 쉽다.

단점:

- 전공군별 탐색이 처음에는 별도 공간처럼 느껴지지 않을 수 있다.
- 향후 콘텐츠가 많아지면 홈이 과밀해질 수 있다.

### 후보 C. 홈 / 전공 / 모집 / 학교 / 프로필

장점:

- 전공 네트워크 정체성이 살아 있다.
- 모집과 학교 보드를 별도로 분리해 의미가 명확하다.
- 알림은 헤더 진입으로 처리 가능하다.

단점:

- 초기에는 홈과 전공의 역할 정의가 중요하다.

## 추천안

MVP에서는 후보 B를 추천한다.

- 홈: 통합 네트워크 피드 + 전공군 필터 + 전공군 집중 보기
- 모집: 스터디/프로젝트/공모전/과제 모집
- 학교: 같은 학교 인증 사용자만 보는 제한형 보드
- 프로필: 내 정보, 인증 상태, 설정

알림은 헤더 우측 아이콘 또는 프로필 내부 진입으로 시작한다. 바텀탭에 넣지 않는다.
전공 전용 탭은 홈 내부 필터 사용량이 충분하고, 각 전공군 피드의 일일 활성 글 수가 안정적으로 유지될 때 V1 이후 재검토한다.

## 추천 네비게이션 구조

### 바텀탭

1. 홈
2. 모집
3. 학교
4. 프로필

### 홈 상단 구조

- 상단 요약 영역
  - 내 전공군
  - 인증 상태
  - 빠른 글쓰기
- 전공군 필터 칩
  - 전체
  - 물리치료
  - 작업치료
  - 방사선
  - 임상병리
- 카테고리 칩
  - 실습
  - 국시
  - 취업
  - 질문답변
  - 자유
  - 모집
- 콘텐츠 섹션
  - 오늘 많이 본 글
  - 최신 글
  - 모집글 미리보기
  - 전공군 집중 보기 진입

## 전체 화면 구조

### 1. 홈

목적:

- 첫 방문 사용자가 앱이 살아 있다고 느끼게 한다.
- 4개 전공군 전체를 묶어 정보 밀도를 확보한다.

핵심 요소:

- 통합 인기글
- 통합 최신글
- 전공군 필터
- 카테고리 필터
- 모집 미리보기

### 2. 전공군 집중 보기

목적:

- 홈 안에서 특정 전공군에 집중된 탐색을 지원한다.

핵심 요소:

- 전공군 선택
- 카테고리별 리스트
- 전공군 설명 또는 자주 묻는 주제
- 홈으로 쉽게 복귀할 수 있는 상태 전환

### 3. 모집

목적:

- 팀모집 수요를 빠르게 연결한다.

핵심 요소:

- 모집 리스트
- 모집 유형 필터
- 전공군 필터
- 모집글 작성

### 4. 학교

목적:

- 같은 학교 학생만 필요한 질문과 소통을 처리한다.

핵심 요소:

- 학교명 표시
- 학교 인증 안내
- 학교 보드 목록 또는 단일 피드
- 학교 한정 글쓰기

### 5. 프로필

목적:

- 내 인증 상태와 기본 설정을 관리한다.

핵심 요소:

- 닉네임
- 전공군
- 학교 인증 상태
- 차단 목록
- 운영 정책/문의 진입

## 초기 MVP 기준 필요한 화면 목록

- 온보딩/인증 진입
- 학교 이메일 인증
- 수동 인증 제출 및 상태 확인
- 홈 피드
- 홈 내 전공군 집중 보기 상태
- 모집 리스트
- 학교 보드
- 글 상세
- 글쓰기
- 모집글 상세/작성
- 신고/차단 액션 시트
- 프로필/설정

## 빈 게시판 문제를 줄이는 IA 장치

### 1. 통합 홈 우선

앱 진입 첫 화면은 전공군별 분리 보드가 아니라 통합 피드여야 한다. 이렇게 해야 초기 콘텐츠가 하나로 모여 "아무것도 없는 앱"처럼 보이지 않는다.

### 2. 통합 인기글과 통합 모집

모집은 전공군별로 쪼개지면 더 빨리 비어 보인다. 모집은 기본적으로 통합 리스트를 제공하고, 전공군 필터로 좁히는 방식이 적합하다.

### 3. 카테고리 우선 탐색

사용자는 "어느 전공인가"보다 먼저 "실습 정보가 필요한가, 국시 정보가 필요한가"로 움직이는 경우가 많다. 따라서 전공군과 카테고리 두 축을 함께 제공해야 한다.

### 4. 학교 보드는 제한형 단일 구조

학교 보드를 여러 개 만들면 모수가 급격히 쪼개진다. MVP에서는 학교별 하나의 보드 또는 아주 적은 수의 구조로 제한한다.

## 화면 간 관계 요약

- 홈은 통합 탐색의 중심이다.
- 전공군 집중 보기는 홈의 세부 상태이며 별도 MVP 탭이 아니다.
- 모집은 홈에서 미리보기가 가능하되 독립 진입점도 가진다.
- 학교는 인증 기반의 제한형 보조 공간이다.
- 프로필은 설정과 운영 관련 액션의 종착점이다.

## 결론

이 앱의 IA는 "학교별 게시판 모음"이 아니라 "전공군 중심 네트워크 + 제한형 학교 보드" 구조여야 한다. MVP에서는 통합 홈과 통합 모집을 중심에 두고, 전공군 탐색은 홈 내부 필터로 해결하는 방식이 가장 단순하면서도 초기 밀도와 정체성을 동시에 만족시킨다.
