# 10. 지표와 분석 이벤트

문서 목적: MVP가 성공하고 있는지 판단하기 위한 핵심 지표와 이벤트 설계 초안을 정의한다.

## 가정 (Assumptions)

- 초기에는 정교한 BI보다 핵심 행동 지표가 중요하다.
- 이벤트 수집 도구는 추후 PostHog 같은 툴을 붙이는 방향을 가정한다.

## MVP 성공 판단 기준

아래 질문에 답할 수 있어야 한다.

1. 사용자가 가입과 인증을 실제로 완료하는가
2. 인증 후 첫 주 안에 첫 글 또는 첫 댓글을 남기는가
3. 홈 피드와 모집이 반복 방문 이유가 되는가
4. 신고/차단이 감당 가능한 수준인가

## 핵심 KPI

| 지표 | 정의 | 이유 |
| --- | --- | --- |
| 가입 시작률 | 앱 설치 후 인증 시작 비율 | 첫 진입 장벽 확인 |
| 인증 완료율 | 인증 시작자 중 인증 완료 비율 | 신뢰 구조의 마찰 확인 |
| 온보딩 완료율 | 인증 완료자 중 닉네임/전공군 설정 완료 비율 | 첫 진입 이탈 지점 확인 |
| 첫 글 작성률 | 가입 후 7일 내 첫 글 작성 비율 | 콘텐츠 공급 측정 |
| 첫 댓글 작성률 | 가입 후 7일 내 첫 댓글 작성 비율 | 상호작용 활성도 측정 |
| D1 / D7 리텐션 | 1일, 7일 재방문 비율 | 반복 방문 가치 확인 |
| 모집 참여율 | 모집 상세 조회자 중 참여 의사 댓글 작성 비율 | 모집 기능 유효성 확인 |
| 학교 보드 사용률 | 인증 사용자 중 학교 보드 방문 비율 | 학교 기능 필요도 확인 |
| 신고 발생률 | 활성 사용자 대비 신고 건수 | 운영 리스크 파악 |
| 제재율 | 신고 중 실제 제재로 이어진 비율 | 정책 강도 점검 |

## 선행 지표

- 홈 피드 조회 수
- 전공군 필터 사용 비율
- 글 상세 진입률
- 댓글 입력 시작률
- 모집글 상세 진입률
- 학교 보드 첫 방문률

## 분석 질문 예시

### 제품

- 어떤 전공군에서 첫 글 작성률이 가장 높은가
- 통합 홈과 홈 내부 전공군 집중 보기 중 재방문에 더 영향을 주는 상태는 무엇인가
- 학교 보드가 실제로 반복 사용되는가

### 운영

- 신고가 많이 발생하는 카테고리는 어디인가
- 수동 인증 반려율이 높은 학교 유형이 있는가

### 성장

- 인증 완료 사용자가 첫 댓글까지 가는 평균 시간은 얼마나 되는가
- 모집글은 어느 전공군에서 가장 반응이 좋은가

## 베타 전 런칭 준비 게이트

아래 기준이 충족돼야 4개 전공군 동시 베타를 여는 쪽이 안전하다.

| 항목 | 최소 기준 | 이유 |
| --- | --- | --- |
| 전공군별 시딩 글 수 | 전공군당 10개 이상, 총 40개 이상 | 첫 진입 빈 피드 방지 |
| 모집 예시 글 | 전체 8개 이상 | 모집 탭 빈 화면 방지 |
| 인증 가능한 학교 수 | 학교 이메일 또는 수동 인증 가이드가 준비된 15개교 이상 | 인증 막힘 최소화 |
| 베타 대기 사용자 | 전공군당 8명 이상, 총 32명 이상 | 4개 전공군 동시 베타 최소 모수 확보 |

## MVP에서 특히 중요하게 볼 지표

초기에는 모든 지표보다 아래 5개를 우선 본다.

1. 인증 완료율
2. 온보딩 완료율
3. 첫 글 작성률
4. 첫 댓글 작성률
5. D7 리텐션

이 다섯 가지가 나쁘면 나머지 기능 확장은 보류하는 편이 낫다.

## 결론

MVP 분석의 핵심은 "인증이 너무 무거운가", "홈이 살아 보이는가", "첫 참여가 일어나는가"를 확인하는 데 있다. 이벤트는 많아 보이더라도, 실제 의사결정에 쓰일 핵심 KPI 중심으로 최소한부터 설계하는 것이 좋다.

## 현재 구현 메모

- 현재 앱은 외부 분석 도구 대신 `AnalyticsProvider` 기반 로컬 이벤트 버퍼를 먼저 사용한다.
- 이미 계측한 핵심 흐름은 인증 시작, 학교 이메일 제출/인증, 학생증 수동 인증 시작/제출/승인/반려, 온보딩 시작/완료, 홈/학교/모집 진입, 글/댓글/모집 생성, 신고/차단이다.
- `manual_verification_started`는 CTA 클릭이 아니라 수동 인증 화면 진입 시점 한 군데에서만 기록한다.
- `major_filter_applied`는 홈 첫 mount가 아니라 사용자가 전공군 필터를 직접 누를 때만 기록한다.
- 다음 단계에서 PostHog 같은 외부 툴을 붙일 때는 이 버퍼의 `track(...)` 진입점을 서버 전송 또는 SDK 호출로 교체하면 된다.

## 다음 단계 후보 이벤트
