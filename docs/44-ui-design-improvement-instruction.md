# 캠퍼스 커뮤니티 앱 UI/디자인 개선 지시서

## 1. 작업 목적

현재 구현된 기능을 유지한 상태에서, 서비스 전반의 디자인 완성도와 UI 사용성을 런치 가능한 수준으로 끌어올린다. 이번 작업의 핵심은 기능 확장이 아니라 `디자인/UI polish`이다.

## 2. 프로젝트 맥락

본 서비스는 보건의료계열 학생 대상 커뮤니티 앱이다.

주요 기능은 다음과 같다.

- 학교 인증
- 학생증 수동 인증
- 전공 보드
- 학교 보드
- 게시글/댓글
- 모집글
- 신고/차단
- moderation 흐름

현재 Supabase 기반 인증 및 검증 흐름이 연결되어 있으며, QA용 시드 데이터도 이미 반영되어 있다.
현 단계는 런치 직전이므로, 기존 동작을 흔들지 않는 선에서 화면 품질을 높이는 것이 최우선이다.

## 3. 작업 범위

다음 항목을 중심으로 UI를 개선한다.

- 전체 화면의 시각적 완성도 향상
- 모바일/웹 공통 레이아웃 정리
- 타이포그래피 개선
- spacing, hierarchy, alignment 정리
- 버튼, 카드, 입력창, 탭, 배지, 상태 UI 개선
- 화면 간 디자인 일관성 강화
- 서비스 성격에 맞는 색상 및 분위기 정리
- 과하지 않은 motion 또는 화면 전환 디테일 보강

## 4. 우선 개선 대상 화면

우선순위는 아래와 같다.

1. 첫 진입 화면
2. 이메일 로그인/회원가입 화면
3. 학생증 수동 인증 화면
4. 홈 피드
5. 전공 보드 / 학교 보드
6. 게시글 상세 / 댓글 UI
7. 모집글 리스트 / 상세
8. 상태 배너, empty state, feedback UI

## 5. 반드시 지켜야 할 제약

다음 사항은 유지해야 한다.

- 기존 기능 흐름 유지
- Supabase auth 로직 임의 변경 금지
- 학교 인증 / 학생증 수동 인증 / verification 흐름 유지
- 보드 id, 데이터 구조, SQL 시드 전제 유지
- 신고/차단/모집/댓글 동작 유지
- 기능 명세 변경보다 UI 중심 개선 우선

즉, 구조를 크게 뒤엎는 리팩터링보다 화면 품질 향상에 집중한다.

## 6. 디자인 방향

디자인은 아래 방향을 따른다.

- 흔한 템플릿형 SaaS 화면처럼 보이지 않도록 할 것
- 대학생 커뮤니티 서비스답게 친근하지만 가볍지 않게 할 것
- 타이포, 간격, 정보 위계를 분명하게 정리할 것
- flat하고 단조로운 화면보다 적당한 깊이감이 느껴지게 할 것
- 모바일에서 먼저 자연스럽고, 웹에서도 어색하지 않게 할 것
- 버튼/입력창/탭/리스트/배지의 상태 구분을 명확히 할 것
- 보건/학업/커뮤니티 맥락에 맞는 차분하지만 생기 있는 색감을 사용할 것
- 과한 효과보다 실제 서비스 같은 안정감 있는 polish를 우선할 것

## 7. 피해야 할 방향

다음 방향은 지양한다.

- 기능 대공사
- 라우팅 구조 대수술
- 데이터 모델 변경
- UI만 화려하고 사용성이 떨어지는 변경
- 실제 서비스처럼 보이지 않는 과한 연출
- 어디서나 본 듯한 보라색 중심 템플릿 느낌
- 기존 인증/권한 흐름을 불안정하게 만드는 수정

## 8. 작업 절차

아래 순서로 진행한다.

1. 코드베이스를 읽고 현재 UI 구조와 화면 흐름을 파악한다.
2. 어떤 화면을 어떤 방향으로 개선할지 간단히 정리한다.
3. 바로 코드 수정까지 진행한다.
4. 수정 후, 어떤 화면이 어떻게 개선되었는지 요약한다.
5. 가능하면 테스트를 수행하거나, 최소한 깨질 가능성이 있는 지점을 점검한다.

## 9. 기대 결과물

최종 결과물은 다음을 포함해야 한다.

- 실제 코드에 반영된 UI 개선
- 핵심 변경 화면에 대한 설명
- 유지한 요소와 변경한 요소의 구분
- 남아 있는 리스크 또는 추가 QA 필요 포인트 정리

## 10. 최종 원칙

이번 작업의 기준은 "기능은 그대로 두고, 런치 가능한 수준의 polished UI로 끌어올리는 것"이다.
기능 안정성을 해치지 않는 범위 안에서, 더 의도적이고 완성도 있는 사용자 경험을 만들어 달라.
